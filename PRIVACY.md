# ky3 Launcher 隐私政策

**最后更新：2026 年 7 月**
**适用版本：6.7.2.0 及以后**
**发布者：KY3 STUDIO**

ky3 Launcher（以下简称"本软件"）是一款开源（MIT 许可证）的原神第三方启动器。我们重视你的隐私。本政策依据软件**实际代码行为**说明本软件收集、存储、使用和传输哪些数据，以及数据流向何处。

> 简要总结：本软件的账号凭据、游戏数据均**仅保存在你本地设备**，仅在你使用相关功能时发送给**米哈游 / HoYoLab 官方服务器**。本软件会向 Sentry 上报崩溃/异常信息，并发送匿名心跳统计与你主动提交的反馈。本软件**不含**广告 SDK，不收集或贩卖用于商业分析的个人画像数据。

## 一、我们收集的数据

### 1. 崩溃与异常诊断数据（通过 Sentry）

本软件集成 Sentry（`Sentry.Extensions.Logging`）用于稳定性诊断。当发生错误（日志级别 Error 及以上）时，会上报以下信息：

- **异常信息**：异常类型、堆栈跟踪、错误码（HRESULT / HTTP / Socket / Win32 错误码）。
- **面包屑（Breadcrumb）**：应用运行过程中 Information 级别及以上的日志记录，可能包含操作步骤、部分文件路径或请求 URL（不含账号凭据）。
- **运行环境**：Windows 版本与 Build 号、WebView2 版本、进程是否以管理员权限运行。
- **设备标识（User.Id）**：见下方"设备标识符"。
- **IP 地址**：应用启动时会查询你的公网 IP，并附加到 Sentry 事件中，用于问题定位与去重。

Sentry 数据发送至 Sentry 官方服务（美国区，`*.ingest.us.sentry.io`）。我们已配置在应用层静默处理的第三方 CDN（如 `cnb.cool`、`seria.moe`）网络错误**不上报**。本软件未启用 Sentry 的"默认个人信息采集（SendDefaultPii）"。

### 2. 设备标识符

- **DeviceId**：由 `你的 Windows 用户名 + 本机 MachineGuid` 经 MD5 哈希生成，作为设备的匿名标识。用于 Sentry 用户标识，以及在向服务器发起请求时通过以下请求头传递：
  - `x-Launcher-device-id`：上述 DeviceId
  - `x-Launcher-device-os`：Windows 版本号
  - `x-Launcher-device-name`：本机计算机名（非 ASCII 字符转义）
- **HoYoLab 设备标识**：为访问米哈游/HoYoLab 接口，会在本次会话内随机生成一组设备标识（如 `x-rpc-device_id` 等），随请求发送给米哈游官方服务器。

### 3. 账号与登录凭据

当你在本软件内登录米游社 / HoYoLab 账号时，会处理并本地存储以下认证信息：

- `LToken`、`SToken`、`CookieToken`（登录令牌 / Cookie）
- 账号 ID（`aid` / `mid`）、绑定的游戏 UID、设备指纹（fingerprint）

**存储位置与方式**：

- 全部保存在本地设备的 SQLite 数据库（`%LocalAppData%\ky3 Launcher\Data\Userdata.db`）的 `users` 表中。
- **登录令牌（`LToken` / `SToken` / `CookieToken`）经 Windows DPAPI 加密后存储**（`ProtectedData`，`CurrentUser` 作用域，Base64 编码），仅能由**同一台设备上的同一 Windows 用户账户**解密。
- 账号 ID（`aid` / `mid`）、游戏 UID、设备指纹（fingerprint）以明文形式存储于同一数据库。

> 说明：应用的普通设置与偏好保存在单独的设置存储中（打包版使用 Windows 应用数据容器；非打包版为 `%LocalAppData%\ky3 Launcher\Settings\LocalSettings.json` 明文 JSON），该设置存储**不包含**上述登录令牌。

**用途与流向**：这些凭据**仅**用于代你访问**米哈游 / HoYoLab 官方 API**（如自动签到、读取便笺/深渊/角色数据、祈愿记录等），**不会**被上传到本软件服务器或任何第三方。

### 4. 心跳与统计

本软件会发送**心跳请求**用于在线状态/活跃度统计，携带**设备标识（DeviceId）、启动器版本、操作系统版本字符串**三项，在每天或每个版本首次启动时发送一次。

### 5. 用户主动提交的反馈

当你在软件内提交反馈时，你填写的**反馈内容**及你选择上传的**图片**会发送到本软件服务器，用于问题处理与回复。此为你主动触发的行为。

### 6. 本地存储的其他数据（不上传）

以下数据仅保存在你的本地设备，用于软件功能，不会主动上传：

- 应用设置与偏好（主题、语言、启动参数、目录配置等）
- 游戏路径、插件启用状态
- 缓存的游戏数据：成就、每日便笺、深渊/幽境危战、角色战斗记录、祈愿记录等（SQLite）
- 图片缓存、游戏截图

## 二、数据发送到的服务器

| 类别 | 服务器 | 用途 |
|------|--------|------|
| 米哈游官方（国服） | `*.mihoyo.com`（passport-api、api-takumi、bbs-api、hyp-api、hk4e-* 等） | 账号登录、签到、游戏数据、公告、祈愿、下载 |
| HoYoverse 官方（国际服） | `*.hoyoverse.com`、`*.hoyolab.com` | 同上（海外版） |
| 本软件服务器 | — | 公告、心跳统计、反馈、公网 IP 查询、元数据/增量补丁、静态资源 |
| 崩溃诊断 | Sentry（`*.ingest.us.sentry.io`，美国区） | 异常与崩溃上报 |
| 角色数据 | `enka.network` | 按游戏 UID 查询角色/装备展示数据 |
| 验证码 | `static.geetest.com` | 登录/签到人机验证 |
| 更新与仓库 | `github.com/ky3-git/ky3-Launcher` | 版本更新、Issue/仓库访问、代理连通性检测 |
| 其他第三方 | 由远程配置下发的镜像/CDN 域名（如 `cnb.cool`、`seria.moe` 等） | 静态资源镜像等非核心服务 |

## 三、我们不做的事

- **不**收集用于广告或商业分析的个人画像数据。
- **不**集成第三方商业分析 SDK（如 Google Analytics、App Insights 等）。
- **不**将你的账号凭据、游戏数据上传至本软件服务器或出售给第三方。
- **不**在后台隐蔽收集与软件功能无关的信息。

---

## 四、数据安全与你的控制权

- 账号凭据与游戏数据存储在你本地设备，你可随时在软件内**退出登录 / 删除账号**，或直接删除数据目录以清除本地数据。
- 你可在系统层面**卸载本软件**以停止所有数据处理。
- 崩溃诊断（Sentry）用于提升稳定性；如你不希望上报，可在网络层面阻断相关域名，或选择不使用本软件。
- 登录令牌经 Windows DPAPI 加密存储，仅能在同一设备的同一 Windows 用户下解密；请勿将数据目录复制到不受信任的环境，并注意保护你的 Windows 账户安全。
